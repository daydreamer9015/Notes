1. 패키지(package)
    1) 자바 파일을 구분하기 위한 폴더로 이해하기
    2) 패키지를 만들었다면, 해당 패키지에 들어가는 자바 파일 첫 줄에 package 패키지명; 과 같이 소속된 패키지명이 선언됨

2. 변수(variable)
    1) 변할 수 있는 값을 저장하기 위한 용도로 사용함
    2) 변수 선언
        - 예) int a;
            - 숫자 정수(integer)를 보관할 수 있는 이름이 a라는 데이터 저장소를 만듦
    3) 변수에 값 대입
        - 예) a = 10;
            - =은 오른쪽에 있는 값을 왼쪽에 저장함
    4) 변수 값 읽기
        - System.out.println(a);    // 변수 a의 값을 읽고 출력
    5) 변수 값 변경
        - 변수는 변할 수 있는 수임
    6) 변수는 반드시 초기화해야함

3. 변수 타입(type)
    1) 변수는 데이터를 다루는 종류에 따라 다양한 형식이 존재함
        - int : 정수를 다룸
        - double : 실수를 다룸
        - boolean : 불리언 타입, true, false 값만 사용 가능함
        - char : 문자 하나를 다루며 작은따옴표를 사용해야 함
        - String : 문자열을 다루며 큰따옴표를 사용해야 함
    2) 리터럴(literal)
        - 고정된 값
        - 정수 리터럴, 실수 리터럴, 불리언 리터럴, 문자 하나 리터럴, 문자열 리터럴

4. 숫자 타입
    1) byte
        - (= 8bit) 표현 길이가 너무 작음
        - 자바는 기본으로 4byte(int)를 효율적으로 계산하도록 설계되어 있음. int 사용함
        - byte 타입을 직접 선언하고 여기에 숫자 값을 대입해서 계산하는 일은 거의 없음
        - 파일을 바이트 단위로 다루기 때문에 파일 전송, 파일 복사 등에 주로 사용됨
    2) short
        - 표현 길이가 너무 작음
    3) int
        - int를 사용함
    4) char
        - 문자 하나를 표현하는 일은 거의 없음
        - 하나의 문자를 표현할 때도 거의 문자열(String)을 사용함
        - 예) String a = "b";
    5) 정수형에서 자주 사용하는 타입
        - int, long
            - 자바는 정수에 기본으로 int를 사용함
            - 20억이 넘을 것 같다면 long을 쓰면 됨
        - 실수
            - double
        - 불린형
            - boolean : true, false
            - 조건문에서 자주 사용함
        - 문자열
            - String

5. 변수 명명 규칙
    1) 전세계 개발자가 해당 관례를 따르기 때문에 사실상 규칙임
        - 변수 이름은 숫자로 시작할 수 없음
            - 숫자를 이름에 포함하는 것은 가능함
        - 이름에는 공백이 들어갈 수 없음
        - 자바의 예약어를 변수 이름으로 사용할 수 없음
        - 변수 이름에는 영문자, 숫자, 달러 기호, 밑줄(_)만 사용함
    2) 클래스는 대문자로 시작, 나머지는 소문자로 시작
        - 예외) 상수는 모두 대문자를 사용하고 언더바(_)로 구분함
               패키지는 모두 소문자를 사용함
    3) 변수 이름은 의미있고, 그 용도를 명확하게 설명해야 함
