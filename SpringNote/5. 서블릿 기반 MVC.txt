1. JSP의 한계
    1) 비즈니스 로직과 View가 섞임 (유지보수 어려움)
    2) Java 코드와 HTML이 혼재 (가독성 저하)
    3) 디자이너와 협업 어려움

2. 권장사항
    1) MVC 패턴 사용 (Model - View - Controller)
    2) JSP는 View 역할만 담당
    3) 비즈니스 로직은 서블릿이나 Service 클래스에서 처리
    4) JSP는 JSTL, EL로 Java 코드 최소화

3. MVC 패턴 적용
    1) 서블릿은 컨트롤러로 사용, JSP는 뷰로 사용해서 MVC 패턴 사용하기
    2) 회원 등록
        - 회원 등록 폼 (컨트롤러)
