* 댓글 기능 - DTO 계층

1. DTO (Data Transfer Object)의 개념
    1) 계층 간 데이터 전송을 위한 객체
    2) Entity와 분리하여 사용
    3) 클라이언트와 서버 간 데이터 교환 형식 정의

2. Entity vs DTO

    구분            Entity                      DTO
----------------------------------------------------------
    목적    데이터베이스 테이블 매핑        데이터 전송
  사용위치   Repository, Service 내부     Controller, View
  변경영향   데이터베이스 스키마 변경        API 스펙 변경
  연관관계   포함 (복잡함)                 단순화 (필요한 정보만)

3. 댓글 기능에서 사용할 DTO
    1) CommentDTO (조회용)
        - 댓글 목록 조회 시 사용
        - Entity -> DTO 변환
    2) CommentCreateDTO (작성용)
        - 댓글 작성 시 사용
        - 클라이언트 -> 서버
        - 유효성 검증 포함
    3) CommentUpdateDTO (수정용)
        - 댓글 수정 시 사용
        - 클라이언트 -> 서버
        - 유효성 검증 포함
