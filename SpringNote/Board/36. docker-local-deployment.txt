* Library Management System - Docker 로컬 배포 실행

1. 환경 정보
    1) 개발 환경
        - OS : Windows 11
        - IDE : IntelliJ IDEA
        - Java : OpenJDK 17
        - Docker : Docker Desktop for Windows
        - Database : MySQL 8.0(로컬 설치)
    2) 프로젝트 정보
        - 프로젝트명 : Library Management System
        - Framework : Spring Boot 3.5.6
        - Build Tool : Gradle 8.5
        - Port : 8081
        - Database : MySQL 8.0

2. 프로젝트 구조
    1) 프로젝트 구조 확인
        - 어플리케이션은 8081 포트 사용
    2) Database 환경
        - MySQL : 3306에서 실행중
        - 데이터베이스 : librarydb
        - 사용자 : root / 비밀번호 : 12345

3. 배포 목표
    1) Docker 컨테이너로 Spring Boot 어플리케이션 실행
    2) 로컬 MySQL 데이터베이스를 그대로 사용 (데이터 보존)
    3) 파일 업로드 디렉토리는 호스트와 공유

4. Docker 배포 프로세스
    1) Phase 1: 준비 단계(Preparation)
        - 프로젝트 구조 분석 및 의존성 확인
        - application.yml 환경 설정 검토
        - 로컬 MySQL 데이터베이스 상태 확인
        - 필수 소프트웨어
            - Docker Desktop 실행
    2) Phase 2: Dockerfile 작성 (Image Definition)
        - Multi-stage Build
            - Stage 1 : Builder
            - Stage 2 : Runtime
    3) Phase 3: .dockerignore 생성 (Build Optimization 최적화)
        - 제외 대상
    4) Phase 4: Docker 이미지 빌드 (Image Build)
        - docker build -t library-management-system .
        - docker images library-management-system
            - 이미지 확인 명령어
    5) Phase 5: Docker Compose 설정
        - docker-compose.yml
        - 실행 명령어
            - docker-compose up -d
        - 실시간 로그 확인 명령어
            - docker-compose logs -f
