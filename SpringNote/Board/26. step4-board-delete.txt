1. 게시글 삭제 기능 구현 플로우

    프런트엔드                         Controller                           Service Layer
------------------------------------------------------------------------------------------------
      STEP1
  사용자가 게시글 상세페이지에서
  삭제 버튼 클릭
        |
       \|/
    확인 대화상자
        |
       \|/
       STEP2                        STEP3
    DELETE /boards/{id} --------> BoardController.delete() --------> 작성자 본인인지 권한 검증
                                                                                |---------------------------------------------
                                                                                |Yes                                         |No
                                                                               \|/                                          \|/
                                                                            게시글 조회
                                                                        boardRepository.findById()
                                                                        + 작성자 정보 (Fetch Join)
                                                                                |
                                                                                |
                                                                               \|/
                                                                              STEP4
                                                                            board.delete()
                                                                             status -> DELETE 변경
                                                                             Dirty Checking가 이루어짐 (UPDATE)
                                                                                |
                                                                               \|/
    redirect:/boards              RedirectAttributes <---------------------- 트랜잭션 커밋
    게시물 목록 페이지로 이동         success 메시지 추가
      + 성공 메시지 표시             "게시글이 삭제되었습니다."

2. 개발 순서 개요
    1) Entity, Repository (데이터베이스)
    2) Service (비즈니스 로직)
    3) Controller (API 엔드포인트)
    4) SecurityConfig (보안 설정)
        - Spring Security의 CSRF 보호 기능 활성화
            - CSRF 활성화 확인
                - 서버 재시작
                - 브라우저. 개발자 도구 열기
                - 게시글 상세 페이지 접속
                - Elements(요소) 탭에서 <head> 확인
    5) layout.html (CSRF 메타 태그)
    6) detail.html (삭제 버튼 UI)
    7) detail.html (JavaScript 삭제 로직)
    8) 테스트 및 검증

3. CSRF
    - layout.html CSRF 메타 태그 추가
        - 모든 페이지에 CSRF 토큰 메타 태그 삽입
