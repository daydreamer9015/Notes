1. 전체 워크플로우 (9단계)
    1) 작업 시작 전 동기화 (git pull)
    2) 작업 브랜치 생성(git checkout -b 브랜치명)
    3) 코드 작업 및 수정
    4) 변경사항 스테이징 (git add .)
    5) 커밋 (git commit -m 메세지)
    6) main 브랜치로 전환 및 동기화 (git checkout main, git pull)
    7) 브랜치 머지 (git merge 해당 브랜치명)
    8) 원격 저장소에 푸시 (git push)
    9) 작업 브랜치 삭제 (git branch -d 브랜치명) <- 옵션
