1. 데이터와 정보
    1) 데이터(Data)
        - 의미 없는 기록 조각
        - 개별적인 사실이나 값
        - 날것(Raw)
    2) 구조화된 데이터(Structured Data)
        - 의미를 담은 재료
        - 최소한의 구조를 가지고 정리됨
    3) 정보(Information)
        - 목적을 가진 결과물
        - 특정한 질문이나 목적을 가지고 분석하고 가공해서 얻어낸 유의미한 결과물
        - 예) 2025년 8월 1일에 가장 많이 팔린 상품은 무엇이고, 총 몇 개나 팔렸는가? -> 이 질문에 대한 답이 '정보'

2. 정보는 왜 중요한가?
    1) 정보를 바탕으로 결정을 내려야 함
        - 의사결정
    2) 흩어져 있는 데이터를 의미있는 정보로 만들 때 비로소 의사결정이 가능해짐
    3) 예
        - 어떤 상품이 잘 팔리는지(정보) 알아야 재고 관리를 효율적으로 할 수 있음
        - 누가 우리 가게의 VIP 고객인지(정보) 알아야 맞춤형 마케팅을 제공할 수 있음
        - 날짜별 매출 추이가 어떤지(정보) 알아야 새로운 프로모션을 기획할 수 있음

3. 데이터베이스
    1) 데이터를 체계적으로 모아놓은 핵심 저장소

4. DBMS 필요한 이유
    1) 컴퓨터 파일 시스템을 이용한 데이터 관리
        - 파일에 직접 저장하고 관리하는 것이 아님
        - 데이터의 중복과 불일치 방지
    2) 데이터 무결성
    3) 동시성 제어
    4) 보안 문제 해결
    5) 데이터 복구 및 백업

5. DBMS 역할과 기능
    1) 데이터 정의 기능(Data Definition Language - DDL)
        - DBMS는 데이터베이스의 구조를 정의할 수 있는 기능
    2) 데이터 조작 기능 (Data Manipulation Language - DML)
        - 일단 데이터베이스 구조가 정의되면 사용자는 이 구조에 맞춰 데이터를 등록(Insert), 조회(Select), 수정(Update), 삭제(Delete)할 수 있어야 함
        - 관계형 DBMS에서는 SQL(Structured Query Language)이라는 표준화된 언어를 주로 사용함
            - ANSI/ISO에서 지정한 SQL 표준

6. RDBMS vs NoSQL
    1) 관계형 데이터베이스 관리 시스템
        - 현재 가장 널리 사용되는 데이터베이스 시스템
        - 주 객체인 테이블 간의 관계를 기반으로 데이터를 구조화
        - 다양한 RDBMS
            - Oracle
            - MySQL : 가장 널리 사용되는 오픈 소스 RDBMS
            - PostgreSQL
            - MS SQL
            - MariaDB(마리아 DB)
            - H2 Database (SQLite)
    2) NoSQL (Not Only SQL)
        - 빅데이터와 특정 요구사항에 맞춰 최근 등장한 시스템
        - 관계형 데이터베이스의 한계를 보완
        - 키-값 (key-value) 저장소, 문서 DB, 
        - 대표 DBMS
            - Redis
            - MongoDB
            - Cassandra

7. 관계형 데이터베이스 핵심 개념
    1) 모든 데이터는 '표'에서 시작함
        - 테이블(Table)
            - 예) 고객 테이블, 주문 테이블, 상품 테이블 등
        - 행(Row)
            - 테이블의 각 가로줄을 의미함
            - 하나의 행은 개별적인 데이터 항목 하나를 나타냄
            - 실무 용어
                - 레코드(Record), 튜플(Tuple)
        - 열(Column)
            - 테이블의 각 세로줄을 의미함
            - 실무 용어
                - 속성(Attribute), 필드(Field)
    2) 기본 키(Primary Key)
        - 수많은 데이터 속에서 '단 하나'를 식별하는 방법
        - 각 데이터를 유일하게 만들어주는 식별자, 기본 키
        - 줄여서 PK라고 함
        - 테이블에 있는 모든 행(Row)들 중에서 특정 행 하나를 유일하게 식별할 수 있는 열(Column) 또는 열들의 조합을 말함
        - 중요한 규칙
            - 고유성(Uniqueness)
                - 절대 중복 불허함
            - Not Null
                - 기본 키로 지정된 열에는 반드시 값이 있어야 함
    3) 외래 키(Foreign Key)
        - 따로 떨어진 '표'들을 관계로 묶는 방법
