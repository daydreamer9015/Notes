1. NOT NULL 제약 조건
    1) 필수 입력 항목

2. UNIQUE 제약 조건
    1) 중복 방지 매커니즘
    2) ORA-00001 : 무결성 제약조건
        - 테이블에 중복된 값을 입력하려 할 때 발생
        - UNIQUE 또는 PRIMARY KEY 제약조건 위반
            - UNIQUE 제약 위반 : UNIQUE 컬럼에 중복 값 입력
            - PRIMARY KEY : 이미 존재하는 기본키 값 입력

3. PRIMARY KEY 제약 조건
    1) 기본키
    2) 인덱스 자동 생성

4. FOREIGN 키 제약 조건
    1) 참조 무결성
    2) 부모-자식 관계 설정

5. CHECK 제약조건, DEFAULT 값
