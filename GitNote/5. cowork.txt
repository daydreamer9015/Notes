1. 협업과 브랜치
    1) 깃허브 원격 저장소 활용
        - 한 장소에서 여러 사람이 함께 작업 가능
            - 브랜치 기능 덕분
                -main branch와 구분짓는 작업을 하기 위해 코드를 분리하는 것 (분기)

    2) 협업 과정
        - 1단계 : 협업을 위한 저장소를 만듦
        - 2단계 : 팀원을 추가함 (공동 작업자)
        - 3단계 : branch별로 commit하고 push함
        - 4단계 : pull request를 요청함 (관리자에게 내가 push한 변경사항을 main에 pull해서 merge해달라고 요청하는 것)
        - 5단계 : Code Review & Code 병합
    
    3) 2단계
        - 프로젝트 관리하기
            - 새로운 프로젝트를 생성
            - 이슈 등록
